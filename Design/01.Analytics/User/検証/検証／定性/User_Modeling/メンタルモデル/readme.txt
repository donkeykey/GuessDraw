詳しいアウトプットとか、その作り方については参考図書を見る。


「メンタルモデル」とは、ユーザーがサイトやタスクをどのようにとらえているかという「頭のなかのモデル」のことを指す。メンタルモデルは従来、認知科学や認知心理学の分野で用いられてきた概念だが、近年デザインの文やでも注目されている。

メンタルモデルと一言で言っても、その内容や形態はさまざまで、対象とするUIや課題、ビジネスゴールなどによって定義する内容は異なる。

最も一般的な所では、「製品やサービスがそのターゲットユーザにどういったかたちで認識されているか」というメンタルモデルがある。たとえば製品に関しての掲示板であっても、ある人にとっては情報源として認識されていたり、ある人にとってはコミュニケーションツールとして認識されていたりというように、ユーザーやその状況によって異なった概念として認識されることがある。

ECサイトやツール的なサイトでは、ターゲットユーザが達成する課題全体をどのように分けて考えているかを示す、「課題分割モデル」というメンタルモデルが用いられる。これはたとえば、「カート機能」と「決済機能」とを別々のタスクとして認識しているか、あるいは１つの大きなタスクとして認識しているかといった部分を指す。この認識が実際の仕組みと合っていないと、システムの振る舞いをユーザが正しく理解することができなくなることが知られている。

また「想起因子／決定因子」というメンタルモデル分析もある。これはたとえばサイト訪問時には「この服に合わせた時計が買いたい」と考えていても、実際の購入決定時には価格と機能で決めてしまっているような、サイト訪問時と実際のアクション時との条件の違いを指す。オクの場合、ユーザはサイト内で実際の選択肢を見ながら検討条件を変えるため、これらの条件は異なっている。この違いを意識するとサイトのトップページで提示する検索方法と、検索や一覧画面での検索条件での項目とは必ずしも同じではなくなる。

ユーザがサイトを長期にわたって使うような場合には学習に寄るメンタルモデルの変化やメタ認知と言った考え方が用いられる。通常ECサイトなどは何度も利用されることが多いが、そういった場合、複数回利用することに寄るユーザーの学習を考慮に入れておかないと、初心者には効果が高いサイトであっても、２回目以降のユーザに対してまどろっこしい印象を与えてしまうことがある。この場合、サイトの特性に応じて、まったく初めての人が多いのかあるいは繰り返し訪れ得る人が多いのかといった主軸を決め、そこに他の観点も加えていく。

またそのサイトでできる全体像辞退を認識できるようになったり、自分がやりたいことをどのようにすればそのサイトで実現できるかを認識できるような能力のことを認識の認識という意味でメタ認知と呼ぶ。通常、メタ認知はユーザーの課題分割能力とも関係しており、課題分割が適切に行えるユーザはメタ認知を迅速に行えるようになり、よりサイトを使いこなしていく。サイトのターゲットユーザの特性を考えるときに、こういったメタ認知を前提にしていくような対応も必要となる。

ユーザーの振る舞いについて、行動だけでなくこういった心理モデルを利用することで、よりサイトの高い効果が見込める。
